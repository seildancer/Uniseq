# Reference Material From Logseq And Related Decisions

This folder captures what Uniseq keeps from Logseq, what it changes, and what must be warned about when a Logseq workspace is opened in Uniseq.

Read in this order:

1. [from-logseq.md](./from-logseq.md) for the short philosophy shift.
2. [logseq-compatibility-contract.md](./logseq-compatibility-contract.md) for the compatibility rules and explicit losses.
3. [logseq-feature-disposition.md](./logseq-feature-disposition.md) for the full feature-family checklist.
4. [framework-decision.md](./framework-decision.md) for the chosen implementation stack.

This folder exists to prevent silent omissions. Every major Logseq feature family should end up as one of:

- kept
- adapted
- separated into a different durable model
- deferred
- dropped
