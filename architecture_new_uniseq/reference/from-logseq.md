# Lessons From Current Logseq

## Keep

- local-first files
- daily journals
- tags and page links
- fast search
- plugin ecosystem mindset
- desktop and mobile ambition
- graph and visual exploration as optional features

## Change

- remove block refs and embeds
- remove always-on block UUIDs
- avoid making the DB abstraction the product center
- make pages projections over authored journals
- make markdown more canonical
- push latency-sensitive work into Rust
- keep focus/zoom as ephemeral UI affordance rather than durable block identity

## Avoid

- exposing too much internal graph structure to users
- requiring users to reorganize captured notes manually
- making sync depend on UI state
- forcing every feature through one database abstraction
