# Settings and Onboarding

Settings and onboarding are first-class shell surfaces.

They are not secondary afterthoughts.

## Settings

Settings should cover:

- workspace configuration
- editor preferences
- calendar and journal preferences
- theme and appearance
- sync status and controls
- plugin management

## Onboarding

Onboarding should help users understand:

- journal-first writing
- page hierarchy
- tags and links as page references
- derived page sections
- optional feature surfaces such as graph or whiteboard

## Design rule

Settings and onboarding should reinforce the core model rather than expose internal implementation vocabulary.
