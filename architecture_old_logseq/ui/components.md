# Component System

Components are organized by feature area rather than by abstract design-system atoms alone.

## Major component groups

- block rendering
- page rendering
- content and editor widgets
- search and query panels
- repo and settings pages
- right sidebar and shell chrome
- onboarding and help
- whiteboard views
- plugin settings and marketplace views

## Design choice

Feature-specific components live near the feature they serve. Shared UI primitives are kept separate.

That makes the code easier to navigate than a purely atomic component tree.
